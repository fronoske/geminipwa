# 手動スモークテスト

## 実施前

- 対象コミット、ブラウザ、OS、画面幅を記録する。
- 既存データを設定画面からエクスポートする。
- 通常プロファイルでは既存データからの起動を、別プロファイルでは初回起動を確認する。
- API送信テストは、原則として無料モデルまたはDummy AIを使用する。

## 起動とデータ互換性

- [ ] 初回起動でチャット画面が表示され、コンソールに未処理エラーがない。
- [ ] 既存プロファイルで設定と会話履歴が維持される。
- [ ] エクスポートしたデータをインポートできる。
- [ ] 再読み込み後も設定と会話履歴が維持される。
- [ ] IndexedDB名 `GeminiPWA_DB` とスキーマバージョン9が意図せず変わっていない。

## チャット

- [ ] 新規チャットを開始できる。
- [ ] ユーザーメッセージとDummy AI応答を保存できる。
- [ ] 過去のチャットを読み込み、編集、再送、複製、削除できる。
- [ ] ストリーミング表示と中断操作が機能する。
- [ ] Geminiが本文なしで終了した場合、終了理由と詳細が空のAI応答バブル内に表示され、再読み込み後も維持される。
- [ ] 全セッションのエクスポート・インポート後もGeminiの `finishReason`、`finishMessage`、`promptFeedback` が維持される。
- [ ] テキスト・画像添付の追加と削除が機能する。
- [ ] Markdown、コードブロック、Mermaid表示に重大な崩れがない。

## 画面と設定

- [ ] チャット、履歴、設定の各画面をボタンとスワイプで移動できる。
- [ ] 設定変更が即時反映または保存操作後に反映される。
- [ ] テーマ、文字サイズ、背景画像、ヘッダー・フッター設定が再読み込み後も維持される。
- [ ] 代表幅360px、768px、1280pxで横方向の意図しないはみ出しがない。
- [ ] キーボード操作時にフォーカス位置が判別できる。

## 補助機能

- [ ] メモ、クリップボードスタック、校正機能が開閉・保存できる。
- [ ] APIプロバイダー切替とAPIキー切替が機能する。
- [ ] 「APIキーの黒塗りを解除する」が有効な状態でAPIキーを切り替えても、通常プロバイダーとLLM Aggregatorのキーが表示されたままになる。
- [ ] 設定と履歴のインポート・エクスポートが機能する。

## PWA

- [ ] Service Workerが登録され、`index.html`、`manifest.json`、`marked.js`、アイコンがキャッシュされる。
- [ ] オンラインで一度起動した後、オフライン再読み込みで画面が表示される。
- [ ] キャッシュ消去操作後に最新版へ更新される。
- [ ] インストール済みPWAから起動でき、表示モードとアイコンが正しい。

## 入力プリセット

- [ ] 設定画面でプリセットを追加・編集・削除でき、再読み込み後も維持される。
- [ ] プリセット本文の改行が入力欄へそのまま挿入される。
- [ ] `{|}`が本文から除去され、その位置へカーソルが移動する。
- [ ] `{|}`がない場合は挿入内容の末尾へカーソルが移動する。
- [ ] 自動送信を有効にしたプリセットが挿入後に送信される。

## Lorebook

- [ ] 初回起動時にサンプルLorebookがIndexedDBへ保存され、設定画面の第1階層「Lorebook」に他のLorebookと同じ形式で表示される。
- [ ] 「新規追加」で全画面エディターへ移動し、設定情報が空の間は「解析」が無効になる。
- [ ] 入力済みのエディターでファイルをロードすると上書き確認が表示され、キャンセル時は入力が維持される。
- [ ] 現在選択中のAPI・モデルで解析され、警告と編集可能な構造化JSONを確認してから保存できる。
- [ ] 原文の文体・視点・会話・表記・禁止事項が`styleGuide`へ、舞台・世界観が固定コアまたは条件付き記憶へ適切に分類される。
- [ ] 「LLMログを表示」で全画面レイヤーが開き、抽出・構造化、原文照合、必要時の構造修復について送信内容・生の応答・エラーを確認できる。
- [ ] LLMログの全画面レイヤーは右上の「×」で閉じられ、スマートフォン幅でもログ領域を十分に使用できる。
- [ ] 解析中は「解析」ボタンが「中断」に変わり、押すと進行中のAPI通信と後続処理が停止する。
- [ ] 解析中にエディター左上の「＜」を押すと中断確認が表示され、キャンセル時は画面と解析が維持される。
- [ ] 解析中にブラウザの戻る操作などでエディターを離れた場合、解析がバックグラウンドで継続しない。
- [ ] すべてのLorebookの「編集」では構造化済みデータが折り畳みフォームで表示され、人物・呼称・条件付き記憶を追加・削除・並べ替えできる。
- [ ] 人物を削除しようとしたとき、その人物を参照する呼称または条件付き記憶があれば削除を止めて参照件数を表示する。
- [ ] 「高度なJSON編集」とフォームを相互に切り替えられ、不正なJSONまたはスキーマ違反がある場合はフォームへ戻らない。
- [ ] スキーマv2のLorebookは読み込み・インポート時にv3へ変換され、旧`characters`条件は`anyCharacters`へ移行される。
- [ ] フォームまたはJSONで保存するとLLMを呼び出さず、元のソーステキストとアプリ管理項目を維持して直接保存する。
- [ ] 初期サンプルを削除して再読み込みしても復活せず、サイトデータを全消去した場合は再投入される。
- [ ] 保存済みLorebookを削除・上下に並べ替えでき、再読み込み後も状態が維持される。
- [ ] 個別・全件エクスポートができ、エクスポートした1件または複数件を新規Lorebookとしてインポートできる。
- [ ] 保存済みLorebookがハンバーガーメニューの選択肢に現れ、既存セッションにも適用できる。
- [ ] 履歴がない初回起動時と「新規チャット」実行時は、選択ダイアログを出さず「使用しない」で開始される。
- [ ] ハンバーガーメニューに現在の「Lorebook：使用しない」が表示される。
- [ ] Lorebook選択に「放課後、夕凪高校で」と「星嶺学園・寮生活日誌」が表示される。
- [ ] どちらかを選ぶと現在のセッションの表示が切り替わり、次の送信から通常のシステムプロンプトと併用される。
- [ ] `lorebook.local.js` を配置した環境では、ローカルLorebookも公開サンプル2冊の後に表示される。
- [ ] 既存セッションでもLorebookを選択・変更でき、「使用しない」へ戻せる。
- [ ] Lorebookを選択したセッションを保存・再読込しても選択状態が維持される。
- [ ] Lorebookを選択したセッションを複製・全出力・全取込しても選択状態が維持される。
- [ ] Lorebook選択をキャンセルした場合、現在のセッションの選択状態が変わらない。
- [ ] 応答中はハンバーガーメニューのLorebook項目が無効になる。
- [ ] 相互に異なる呼称を持つ二人が登場する送信では、APIリクエストのLorebook参照に方向別の呼称がそれぞれ含まれる。
- [ ] 発話と内心で呼称が異なる二人が登場する送信では、それぞれの呼称が文脈別に含まれる。
- [ ] OpenRouterの最新モデル一覧を取得してから応答を生成し、AI応答の操作欄に「合計トークン数 / モデルのコンテキスト上限 (使用率 %)」が表示される。
- [ ] Geminiではモデル情報APIから入力トークン上限が自動取得され、同じ形式で使用率が表示される。
- [ ] コンテキスト上限を取得できないモデルでは、割合を捏造せず「合計トークン数 / 上限不明」と表示される。
- [ ] 1,000以上のトークン値は、値の大きさに応じて `K`、`M`、`G` で短縮表示される。
- [ ] コンテキスト使用率が90%以上の表示は警告色になる。

## 入力・出力移動ボタン

- [ ] 上矢印で前、下矢印で次のメッセージ末尾へ移動する。
- [ ] 上矢印は先頭メッセージ末尾の次に画面最上部へ移動し、最上部では無効になる。
- [ ] 入力欄が複数行になっても、ボタンが入力欄の上に表示される。
- [ ] 「常に表示」「非表示」「スクロール時のみ表示」が再読み込み後も維持される。
- [ ] 「スクロール時のみ表示」ではスクロール開始時に現れ、停止後に隠れる。

## チャット入力欄

- [ ] 入力欄が初期状態で2行の高さになる。
- [ ] 添付「＋」ボタンが「送」ボタンの上に表示される。
- [ ] 入力欄の上端が「＋」ボタン、下端が「送」ボタンの端と揃う。
- [ ] フッターのAPI切替・APIキー切替・貼付けボタンを複数表示すると、縦に並ぶ。
- [ ] 履歴ボタンが時計型の履歴アイコンで表示される。
- [ ] 送信中に別チャットを選ぶと、中断確認ダイアログが表示される。
- [ ] 「アプリの挙動を調整」と「表示の調整」が同じ階層に表示される。
- [ ] テーマ、レイアウト、ボタン、フォント、透明度の設定が「表示の調整」にまとまっている。

## ヘッダーメニュー改修後に追加する確認

- [ ] 「新」「削」「コ」がヘッダーに表示されない。
- [ ] ハンバーガーメニューから「新規チャット」「全削除」「全コピー」を実行できる。
- [ ] メニューは外側タップ、Escape、項目選択で閉じる。
- [ ] ハンバーガーボタンの表示設定が再読み込み後も維持される。
- [ ] 「全削除」で先頭項目だけが残る。
